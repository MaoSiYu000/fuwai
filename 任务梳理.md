# 基于多源数据的大学生行为分析与干预模型设计 — 任务梳理

## 一、项目背景与目标

### 问题说明

- 高校数字化校园积累了**海量多源异构**学生行为数据，可支撑学业影响因素分析与成效预测。
- 传统研究依赖问卷调查、小样本实验，存在**滞后性、主观性强**等问题，难以全面揭示学业影响的复杂机制。
- 目标：基于学生**客观行为痕迹**，挖掘认知与学业发展规律，推动智慧教育从**“事后补救”**向**“事前预警、过程干预”**转型。

### 用户期望（五大方向）

1. **基于多源数据的学生画像生成与更新**
2. **学业轨迹演化模式挖掘与关键行为分析**
3. **面向教育大数据可视化叙事模型设计**
4. **学业风险动态感知与预警模型构建**
5. **基于归因解释与知识增强提示的个性化报告生成**

### 开发说明（核心产出）

- 对多源数据进行**关联分析与数据挖掘**，建立**学生行为分析模型**。
- 输出**学习行为数据分析成果**，并生成**个性化评价报告**。

### 技术要求与指标（必达）


| 指标     | 要求                               |
| ------ | -------------------------------- |
| 分析成果数量 | 8~10 个学生行为数据分析成果                 |
| 报告内容   | 必须包含**学生个体画像**和**群体画像**，其他成果可自定义 |
| 模式发现   | 至少发现学生的 **4 类模式**                |
| 预测模型   | 相关预测模型 **AUC ≥ 80%**             |
| 可解释性   | 对关键问题提供 **不少于 3 个维度** 的解释        |

## 二、样本数据范围（多源数据）

项目可用的样本数据包括（按类别梳理）：


| 类别   | 数据表/内容                                      |
| ---- | ------------------------------------------- |
| 学生基础 | 学生基本信息、一卡通账户信息                              |
| 学习行为 | 在线学习平台访问记录、线上学习数据、作业提交记录、考试提交记录、课堂任务参与、讨论记录 |
| 学业结果 | 成绩记录、综合测评、选课信息                              |
| 校园生活 | 一卡通交易流水、上网日志明细、读者行为、图书馆借阅记录与历史、跑步打卡数据       |
| 发展结果 | 奖学金获奖信息、学籍异动、就业信息等                          |


当前项目已有部分 CSV（如：学生基本信息、成绩、选课、签到、作业提交、线上学习综合表现、学籍异动、学科竞赛、体能考核等），后续可随赛题开放逐步补充一卡通、上网日志、图书馆、跑步等数据。

## 三、分阶段任务拆解

### 第一阶段：数据准备与清洗（当前阶段）


| 序号  | 任务        | 说明                                                                     |
| --- | --------- | ---------------------------------------------------------------------- |
| 1.1 | 数据接入与格式统一 | 将各源数据（Excel/CSV 等）统一为可分析格式，存入 `data/`，原始表结构、字段含义整理到《字段说明》文档            |
| 1.2 | 数据清洗      | 在 `code/` 下编写清洗脚本：缺失值、异常值（如成绩超满分、非法编码）、重复记录、时间格式标准化等，清洗结果输出到 `output/` |
| 1.3 | 清洗报告与校验   | 记录每个表清洗前后行数、删除比例，避免过度删除；对关键表做抽样检查，保证后续分析可用                             |


**产出**：`data/` 下的规范数据、`output/` 下的清洗后数据、`code/` 下可复用的清洗脚本、清洗统计报告。

---

### 第二阶段：多源数据关联与特征工程


| 序号  | 任务            | 说明                                                                   |
| --- | ------------- | -------------------------------------------------------------------- |
| 2.1 | 主键与实体统一       | 确号 LOGIN_NAME），建定跨表主键（如学号 XH、账立「学生」统一 ID，处理学号与账号的映射关系                |
| 2.2 | 多表关联与宽表/主题表构建 | 按分析主题关联多张表（如：学生 × 成绩 × 选课 × 作业 × 签到），形成「学业行为主题表」「生活行为主题表」等，便于后续画像与建模 |
| 2.3 | 特征工程          | 从原始字段衍生特征：如出勤率、作业按时提交率、成绩趋势、选课负荷、图书馆/上网/跑步等行为统计；为画像和预警模型准备特征矩阵       |


**产出**：统一学生 ID 映射表、若干主题宽表或特征表、特征说明文档。

---

### 第三阶段：学生画像（个体 + 群体）— 必须产出


| 序号  | 任务        | 说明                                                                                                         |
| --- | --------- | ---------------------------------------------------------------------------------------------------------- |
| 3.1 | 个体画像生成与更新 | 基于多源数据为每位学生生成结构化画像：基础属性（院系、专业、年级）、学业表现（成绩、绩点、排名）、学习行为（出勤、作业、在线学习、讨论参与）、生活行为（一卡通、图书馆、运动等）；设计画像更新逻辑（如按学期/按月） |
| 3.2 | 群体画像      | 按院系、专业、年级、班级等维度做群体统计与对比：成绩分布、行为分布、风险比例等；与个体画像一起作为「个性化评价报告」的必备内容                                            |
| 3.3 | 画像存储与接口   | 画像结果可落表或 JSON，供报告生成与可视化调用                                                                                  |


**产出**：个体画像模型/表、群体画像统计结果、画像更新脚本或 pipeline。

---

### 第四阶段：学业轨迹与关键行为分析


| 序号  | 任务         | 说明                                                                          |
| --- | ---------- | --------------------------------------------------------------------------- |
| 4.1 | 学业轨迹演化模式挖掘 | 按时间维度刻画学生学业轨迹（如成绩变化、选课序列、作业提交习惯演变），用序列分析、聚类等方法发现典型轨迹模式（如「持续优秀」「先低后高」「下滑型」等） |
| 4.2 | 关键行为分析     | 识别与学业成效强相关的行为指标（如出勤率、作业提交率、在线学习时长、图书馆借阅等），做相关性分析、回归或因果推断，为预警和解释提供依据         |
| 4.3 | 模式归纳       | 至少提炼 **4 类学生模式**（如学业型、社交型、风险型、均衡型等），与赛题「至少 4 类模式」要求对齐                       |


**产出**：轨迹模式分类结果、关键行为清单及与成绩/风险的关系分析、4 类及以上模式定义与标签。

---

### 第五阶段：学业风险动态感知与预警模型


| 序号  | 任务      | 说明                                                                                |
| --- | ------- | --------------------------------------------------------------------------------- |
| 5.1 | 风险定义与标注 | 定义学业风险（如挂科、绩点骤降、学籍异动、退学倾向等），利用成绩、异动、行为数据构造正负样本或等级标签                               |
| 5.2 | 预警模型构建  | 选用分类模型（如逻辑回归、随机森林、XGBoost 等）对风险进行预测，**确保 AUC ≥ 80%**；考虑时序特征，实现「动态」感知（如按学期/按月更新预测） |
| 5.3 | 模型评估与迭代 | 划分训练/验证/测试集，报告 AUC、精确率、召回率等；若未达 80%，通过特征、样本、模型调参进行迭代                              |


**产出**：风险定义文档、预警模型（含训练代码与权重/配置）、达到 AUC≥80% 的评估报告。

---

### 第六阶段：可视化叙事模型设计


| 序号  | 任务     | 说明                                                   |
| --- | ------ | ---------------------------------------------------- |
| 6.1 | 叙事逻辑设计 | 围绕「个体—群体—风险—行为」设计故事线：如从群体概览到个体详情，从现状描述到风险预警，从行为到归因   |
| 6.2 | 图表与看板  | 选择并实现图表类型（分布图、趋势图、雷达图、桑基图等），制作仪表盘或叙事看板，支持教育大数据的故事化呈现 |
| 6.3 | 交互与导出  | 支持筛选（院系、年级、个人）、下钻；关键图表可导出或嵌入报告                       |


**产出**：可视化叙事方案文档、看板/大屏或 Web 可视化页面。

---

### 第七阶段：归因解释与个性化报告生成


| 序号  | 任务      | 说明                                                                     |
| --- | ------- | ---------------------------------------------------------------------- |
| 7.1 | 归因与可解释性 | 对关键问题（如为何被预警、成绩为何波动）提供**不少于 3 个维度**的解释，可结合：特征重要性、SHAP/林德利、规则抽取、自然语言描述等 |
| 7.2 | 知识增强与提示 | 利用领域知识（如培养方案、课程难度、历史规律）增强解释或生成提示语，使报告更贴近教育场景                           |
| 7.3 | 个性化报告生成 | 整合个体画像、群体对比、轨迹与模式、风险预警、归因解释，生成**个性化评价报告**（可自动化模板 + 关键指标与文案生成）          |


**产出**：归因与解释方法说明、报告模板、报告生成脚本或服务，以及示例报告。

---

### 第八阶段：成果整合与达标自检


| 序号  | 任务           | 说明                                                                        |
| --- | ------------ | ------------------------------------------------------------------------- |
| 8.1 | 8~10 个分析成果清单 | 明确列出 8~10 个「学生行为数据分析成果」（如：个体画像、群体画像、轨迹模式、关键行为分析、风险预警、某几类行为专题分析等），确保覆盖赛题要求 |
| 8.2 | 指标自检         | 核对：① 报告是否包含个体+群体画像；② 是否至少 4 类模式；③ 预测模型 AUC≥80%；④ 关键问题≥3 维解释               |
| 8.3 | 文档与答辩准备      | 整理技术文档、使用说明、演示数据与答辩材料                                                     |


**产出**：成果清单、自检表、技术文档与答辩材料。

## 四、推荐执行顺序（总览）

```
阶段一 数据准备与清洗
    ↓
阶段二 多源数据关联与特征工程
    ↓
阶段三 学生画像（个体+群体）  ← 报告必备
阶段四 学业轨迹与关键行为分析 → 至少 4 类模式
阶段五 学业风险预警模型       → AUC≥80%
    ↓
阶段六 可视化叙事模型
阶段七 归因解释与个性化报告   → ≥3 维解释
    ↓
阶段八 成果整合与达标自检     → 8~10 个成果
```
