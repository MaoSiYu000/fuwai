# 聚类前准备（Pre-cluster Prep）清单与操作说明

> 目的：在正式聚类前，把特征“做稳/做干净/可复现”，提升聚类稳定性与画像可解释性。  
> 原则：**尽量多用信息**；能对齐学期的做“学期级”，对齐不了的做“总体级”；处理过程与产出与其它模块隔离。

---

## 1. 目录约定（隔离）

- **代码目录**：`code/cluster_prep/`
- **输入**：优先使用 EDA 产出的草稿特征表  
  - `output/eda/features_student_term_draft.csv`
- **输出目录**：`output/cluster_prep/`

---

## 2. 输入数据结构（两类特征）

### 2.1 学期级特征（student-term）

- 键：`XH + TERM_KEY`
- 例：成绩（学期）、体能考核（学期）、体测年度保守映射（学期）、学籍异动（日期→学期）、竞赛（日期→学期）等

### 2.2 总体级特征（student-level）

- 键：`XH`
- 例：四六级最高分/级别、综合测评总体排名、奖学金总体次数/金额等  
- 合并方式：把总体级特征合并到该学生所有 `TERM_KEY` 行上（用于聚类/画像解释）

---

## 3. 聚类前准备清单（按顺序，模块化脚本对应）

> 说明：每一步都在 `output/cluster_prep/` 产出新文件，避免覆盖原始数据。

### 3.1 Step 00：复制输入特征表（固定版本，便于回溯）

- **目的**：把 EDA 草稿特征表“冻结一份”作为本次聚类前处理输入
- **脚本**：`00_freeze_input.py`
- **输出**：
  - `00_features_frozen.csv`

### 3.2 Step 01：缺失处理（缺失指示 + 保守填补）

- **目的**：缺失本身是一种模式，不能简单当 0
- **做法**：
  - 关键特征生成 `is_missing_*` 指示列
  - 数值缺失先按“同列中位数”填补（保守、稳定），同时保留缺失指示
- **脚本**：`01_missing_indicators_and_impute.py`
- **输出**：
  - `01_features_imputed.csv`

### 3.3 Step 02：稳健化（winsorize / robust scaler）

- **目的**：长尾/极端值会绑架距离度量，影响 KMeans/GMM
- **做法**：
  - 对指定列做 winsorize（默认 p1–p99）
  - 之后做 robust scaler 或标准化（z-score）
- **脚本**：`02_robustify_scale.py`
- **输出**：
  - `02_features_scaled.csv`
  - `02_scale_params.json`（记录分位数/缩放参数，便于复现）

### 3.4 Step 03：冗余特征处理（相关性/语义二选一）

- **目的**：强相关特征会重复计权，导致簇被某维度“放大”
- **做法**：
  - 相关系数 > 阈值（默认 0.90）做成候选对
  - 默认只删“更次要/更难解释”的一列（可在配置里改）
- **脚本**：`03_drop_redundant.py`
- **输出**：
  - `03_features_pruned.csv`
  - `03_dropped_features.csv`

### 3.5 Step 04：异常点标注（不急着删，先标注）

- **目的**：先识别“异常学生-学期”，用于后续决定是否需要 HDBSCAN/噪声处理
- **做法**：对关键列按分位数阈值标注 `is_outlier_*`
- **脚本**：`04_flag_outliers.py`
- **输出**：
  - `04_features_outliers_flagged.csv`
  - `04_outlier_summary.csv`

### 3.6 Step 05：降维只用于检查（可选）

- **目的**：PCA 仅用于判断是否“被1-2个主成分支配”，不直接替代可解释特征
- **脚本**：`05_pca_check.py`
- **输出**：
  - `05_pca_explained_variance.csv`

---

## 4. 最终给聚类算法的输入

- **推荐输入文件**：`output/cluster_prep/04_features_outliers_flagged.csv`
- 保留列：`XH, TERM_KEY` + 处理后的数值特征 + 缺失指示 + 异常标注

---

## 5. 运行方式（一次跑完）

在项目根目录：

```bash
python code/cluster_prep/run_all.py
```
