# 当前模式说明（通俗版，已更新到 mode=10）

## 1. 大类（mode）和小类（subtype）怎么分？

### 1.1 大类 mode（先分）

- 每条“学生-学期”记录先压成 6 个稳定维度分数：
  - `dim_academic`（学业）
  - `dim_attendance_engagement`（出勤/参与）
  - `dim_homework_behavior`（作业行为）
  - `dim_online_learning`（线上学习）
  - `dim_fitness`（体能）
  - `dim_development`（发展成就：奖学金/竞赛/四六级）
- 再用 GMM 在这些维度分数上做软聚类，得到 mode（大类）。
- 本轮候选 K 调到 `4..12` 后，按 BIC 自动选出 **mode=10**。

### 1.2 小类 subtype（再分）

- 在每个 mode 内，再用更细的原始特征（冻结表）做一次 GMM 软聚类。
- 用来把同一大类里不同风格的人再拆开，便于解释。
- 当前每个 mode 下是 4 个 subtype（总计 40 个 subtype）。

### 1.3 subtype_id 怎么看？

- 编码规则：`subtype_id = mode_id * 100 + 子类编号`
- 例如：`203` = `mode 2` 下 `sub 3`；`901` = `mode 9` 下 `sub 1`

- `mode 5`：最明显的高风险波动群（学业显著偏低、作业行为波动更大）。

- `mode 9`：主流均衡群体，可作为基准组对比。
