# 聚类前准备（Pre-cluster Prep）清单与操作说明

> 目的：在正式聚类前，把特征“做稳/做干净/可复现”，提升聚类稳定性与画像可解释性。  
> 原则：**尽量多用信息**；能对齐学期的做“学期级”，对齐不了的做“总体级”；处理过程与产出与其它模块隔离。

## 0. 先给你一个“现阶段总结”（你关心的）

### 0.1 我们目前做了什么处理？

- **冻结输入**：把 `output/eda/features_student_term_draft.csv` 复制为 `00_features_frozen.csv`，保证本次处理有“固定版本”可复现
- **缺失处理**：为缺失生成 `is_missing_*` 指示列；数值缺失用**中位数**保守填补（同时保留缺失指示）
- **稳健化**：对长尾特征做 winsorize（p01–p99）；再做 RobustScaler（median/IQR）缩放，减少极端值对距离的影响
- **低方差剔除**：删除零方差/近零方差特征（对聚类没区分度）
- **方差均衡（防支配）**：把各特征再做一次“单位方差缩放”，避免距离/PCA 被少数指标支配
- **去冗余**：对强相关特征（默认阈值 0.90）进行“二选一”删除，避免重复计权
- **异常点先标注不删除**：按 p01/p99 给每个特征打 `is_outlier_lo_* / is_outlier_hi_*` 标签，后续用于决定是否需要 HDBSCAN/噪声策略
- **PCA 仅作诊断**：用 PCA 看“有没有少数方向主导方差/数据是否近似低维结构”，不直接替代最终画像特征

### 0.2 我们得出了什么结果（本次跑完的数字结论）？

- **粒度**：学生-学期（`XH + TERM_KEY`）
- **规模**：rows=20014
- **特征数变化**：
  - 冻结后：cols=36
  - 加缺失指示后：cols=70
  - 低方差剔除：删除 3 个特征（方差=0：`sch_cnt_term`、`att_abnormal_rate`、`yd_type_nunique`）
  - 去冗余后：删除 1 个特征（`kccj_median`）
  - 加异常标记后：新增 66 列异常标签
- **异常点图里“很多 0”是正常现象**：因为我们用的是“严格 `<q01` / `>q99`”判定；离散/扎堆/被截断后的特征可能会出现大量 0（见本文件 Step 04 解释与 `04_outlier_summary.csv`）
- **PCA 诊断（做方差均衡后）**：PC1≈0.179，PC2≈0.083，前两维累计≈0.261（更符合“多维结构”，不再被 1–2 个指标支配）

### 0.3 下一步怎么做？先聚类还是继续完善准备？

结论：**建议先做 2 个“短平快补强”，然后就进入聚类（不要无限前处理）。**

- **先补强（1–2 次迭代就够）**：
  - **补强 A：处理“零方差/近零方差”特征**（它们对聚类无贡献、但会干扰一些统计/可视化）
  - **补强 B：检查 PCA 主导来源**（找出贡献最大的 3–5 个特征：是否因为 IQR 过小导致缩放后被放大；必要时对少数特征做更保守的缩放或改用 `QuantileTransformer` 仅用于聚类输入）
- **然后进入聚类**：
  - 先用 **KMeans（基线）+ GMM/LPA（对照）** 做多 K 训练
  - 用 **稳定性（ARI）+ 可解释性（簇画像差异）** 来定 K 和模型
  - **HDBSCAN** 仅作为“异常/噪声结构”补充，而不是一上来就替代全部聚类

## 3. 聚类前准备清单（按顺序，模块化脚本对应）

> 说明：每一步都在 `output/cluster_prep/` 产出新文件，避免覆盖原始数据。

### 3.1 Step 00：复制输入特征表（固定版本，便于回溯）

- **目的**：把 EDA 草稿特征表“冻结一份”作为本次聚类前处理输入
- **脚本**：`00_freeze_input.py`
- **输出**：
  - `00_features_frozen.csv`

### 3.2 Step 01：缺失处理（缺失指示 + 保守填补）

- **目的**：缺失本身是一种模式，不能简单当 0
- **做法**：
  - 关键特征生成 `is_missing_*` 指示列
  - 数值缺失先按“同列中位数”填补（保守、稳定），同时保留缺失指示
- **脚本**：`01_missing_indicators_and_impute.py`
- **输出**：
  - `01_features_imputed.csv`

### 3.3 Step 02：稳健化（winsorize / robust scaler）

- **目的**：长尾/极端值会绑架距离度量，影响 KMeans/GMM
- **做法**：
  - 对指定列做 winsorize（默认 p1–p99）
  - 之后做 robust scaler 或标准化（z-score）
- **脚本**：`02_robustify_scale.py`
- **输出**：
  - `02_features_scaled.csv`
  - `02_scale_params.json`（记录分位数/缩放参数，便于复现）

### 3.3b Step 02b：删除零方差 / 近零方差特征（缩放后）

- **目的**：删掉“所有人几乎都一样”的列（对聚类没区分度，也会干扰诊断/可视化）
- **脚本**：`02b_drop_low_variance.py`
- **输出**：
  - `02b_features_var_pruned.csv`
  - `02b_dropped_low_variance.csv`

### 3.3c Step 02c：方差均衡（避免少数指标支配）

- **目的**：让各特征的“波动尺度”更接近，避免聚类/ PCA 几乎只按某 1–2 个指标分组，从而更容易形成“多维画像”
- **做法**（脚本实际干了什么）：
  - 输入是缩放后的特征表 `02b_features_var_pruned.csv`
  - 对每个数值特征计算标准差 `std`
  - 再做一次“单位方差缩放”（只作用于聚类输入空间）：
    - \(x_{new} = x / std\)
  - 这样每个特征缩放后标准差大约都接近 1（更公平），距离度量就不会被“std 超大”的特征牵着走
- **脚本**：`02c_equalize_variance.py`
- **输出**：
  - `02c_features_std_equalized.csv`
  - `02c_std_params.csv`

### 3.4 Step 03：冗余特征处理（相关性/语义二选一）

- **目的**：强相关特征会重复计权，导致簇被某维度“放大”
- **做法**：
  - 相关系数 > 阈值（默认 0.90）做成候选对
  - 默认只删“更次要/更难解释”的一列（可在配置里改）
- **脚本**：`03_drop_redundant.py`
- **输出**：
  - `03_features_pruned.csv`
  - `03_dropped_features.csv`

### 3.5 Step 04：异常点标注（不急着删，先标注）

- **目的**：先识别“异常学生-学期”，用于后续决定是否需要 HDBSCAN/噪声处理
- **做法**（脚本实际干了什么）：
  - 对每个数值特征计算分位数阈值：p01 与 p99
  - 生成两列标记：
    - `is_outlier_hi_<col>`：该行该列值 **> p99** 记 1，否则 0
    - `is_outlier_lo_<col>`：该行该列值 **< p01** 记 1，否则 0
  - 注意：这里用的是 **严格大于/小于**（不是 `>=` / `<=`），因此对“离散/大量相同值”的特征，异常率可能为 0（这是预期现象）
- **脚本**：`04_flag_outliers.py`
- **输出**：
  - `04_features_outliers_flagged.csv`
  - `04_outlier_summary.csv`

### 3.6 Step 05：降维只用于检查（可选）

- **PCA 是什么？（通俗解释）**
  - **PCA（Principal Component Analysis，主成分分析）**：把很多个相关的特征“压缩”成少数几个新的综合方向（主成分）。
  - 它做的事是：找一个方向，让数据在这个方向上的**方差最大**；再找第二个方向，要求与第一个方向垂直，依次类推。
  - **我们在这里用 PCA 的目的**：纯诊断——看看数据是不是“本质上接近低维”、或者“被少数特征/少数方向主导”。  
    它**不用于**最终画像（因为主成分不好解释，做画像不直观）。
- **目的**：PCA 仅用于判断是否“被 1–2 个主成分支配”，不直接替代可解释特征
- **脚本**：`05_pca_check.py`
- **输出**：
  - `05_pca_explained_variance.csv`
