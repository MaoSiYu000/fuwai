# 数据集说明与处理规范

本文档记录各 CSV 数据集的**字段特性、数据问题、处理方式及关联关系**，便于数据清洗与后续建库、建模使用。按表逐个整理，一个文件夹（一张表）一个章节。

---

## 1. 学生作业提交记录（学生作业提交记录.csv）

### 1.1 数据概览与用途

- **粒度**：单条记录 = 某次作业/考试的一次提交（按创建人/教学班/课程/作业等维度可聚合）。
- **用途**：分析作业提交量、按时完成情况、得分与满分关系、做作业耗时；与成绩、选课、教学班等表关联做学业行为分析。

### 1.2 字段说明与特性

| 字段名 | 说明 | 特性与注意 |
|--------|------|------------|
| **CREATER_ID** | 创建人 id | 与提交记录对应；**个数可反映该创建人对应的提交次数**（次数越多≈提交的作业数越多）。 |
| **CREATER_NAME** | 创建人姓名 | **均为「同学\|老师」等脱敏值，无真实姓名**；分析时忽略，身份以账号为准。 |
| **CREATER_LOGIN_NAME** | 创建人账号 | 与 CREATER_ID 一一对应，**即学号/登录账号**（如 pjxyqwbk736）。与体能考核、基本信息等表的 **XH（学号）为同一标识**，是跨表关联学生的关键字段。 |
| **COLLEDGE_ID** | 院系 id | 学院 ID，可统计**某学院的作业提交情况**。同一院系名称可能对应多个 ID（如 10960618 与 628413 均可能为「通信工程学院」）。 |
| **COLLEDGE_NAME** | 院系名称 | 一个院系名称可能对应多个 COLLEDGE_ID，关联时以 ID 为准或做院系统一映射。 |
| **MAJOR_ID** | 专业 id | 存在 **0**；若为 0 则 **MAJOR_NAME 通常为空（空格）**，视为缺失，需考虑填补（见下文处理）。 |
| **MAJOR_NAME** | 专业名称 | 同一专业名可能对应多个 MAJOR_ID（不同院系 ID 下）。 |
| **CLASS_ID** | 班级 id | 存在 **0**；若为 0 则 **CLASS_NAME 一定为空**。同一班级名可能对应多个 CLASS_ID。 |
| **CLASS_NAME** | 班级名称 | 格式不统一：有「2111051」「2003051嵌」等（如软工嵌入式/网络方向）；也有「班级2TS00602924」「班级1PY01400304」等。 |
| **REVIEW_ID** | 批阅人 id | 很多为 0，**可不做重点处理**。 |
| **REVIEW_NAME** | 批阅人姓名 | 多为「同学\|老师」，可不处理。 |
| **REVIEW_LOGIN_NAME** | 批阅人账号 | 很多为空，**可不做重点处理**。 |
| **WORK_ID** | 作业 id | 基本唯一，可作为作业维度主键。 |
| **WORK_TITLE** | 作业标题 | 可做**词意/关键词提取**，用于作业类型、难度等后续分析。 |
| **CLAZZ_ID** | 教学班 id | 格式较固定。 |
| **CLAZZ_NAME** | 教学班名称 | 格式不统一（班级2TS00602924、2111051 等混在一起），清洗时可标准化或仅保留 ID。 |
| **COURSE_ID** | 课程 id | 与选课、成绩等表关联的关键字段。 |
| **COURSE_NAME** | 课程名称 | 一个课程名可能对应多个 COURSE_ID。 |
| **STATUS** | 提交/批阅状态 | 3=待批阅，4=已批阅，0/1=未做；**当前数据中仅出现 3 和 4**。 |
| **FULLMARKS** | 满分 | 数值，需保证 SCORE ≤ FULLMARKS（异常则删除或改为满分）。 |
| **COMMENT** | 批语 | 结构混乱，多为空；部分为 HTML（如 `<p>无英文摘要，无致谢。</p>`），**后期可解析 HTML 提取正文**。 |
| **SCORE** | 分数 | **必须 ≤ 满分**；为 0 可能是未批阅，**算学生平均分时需排除未批阅（如 STATUS≠4 或 SCORE 为 0 且未批阅）**。 |
| **TYPE** | 类型 | 1=作业，2=考试；当前表内均为 1。 |
| **CREATER_TIME** | 作业创建时间 | 与 ANSWER_TIME 相减可得到**做作业耗时**（需统一时间格式）。 |
| **ANSWER_TIME** | 回答时间 | 重要；与 CREATER_TIME 一起算作答时长。 |
| **REVIEW_TIME** | 批阅时间 | 可选分析。 |
| **UPDATE_TIME / INSERT_TIME / TSTAMP** | 更新时间/插入时间/时间戳 | 技术字段；**重点用 CREATER_TIME、ANSWER_TIME**。 |

**时间格式注意**：  
- 有的为「0:59:00」「0:02:00」这种仅时间（不足 1 小时），需结合日期判断先后（如 14:59 发布、15:02 提交）。  
- 有的为「163-03-18 05:30:00.0」等完整日期时间（可能是 2021-03-18 的误写）。  
- 清洗时需**统一解析为 datetime**，再计算作答时长；跨天或异常间隔需定规则（如截断为 24 小时内或标为缺失）。

### 1.3 数据问题与处理方式

| 问题 | 处理方式 |
|------|----------|
| **MAJOR_ID=0 且 MAJOR_NAME 为空** | 视为缺失。可：① 用其他表（如学生基本信息、选课）按 CREATER_LOGIN_NAME/院系补专业；② 或用同院系（COLLEDGE_ID）下**众数专业**填补。 |
| **CLASS_ID=0 且 CLASS_NAME 为空** | 同上，可从他表按账号/院系补班级，或用院系内众数班级填补。 |
| **SCORE > FULLMARKS** | 异常。**删除该行**或将该行 SCORE 改为 FULLMARKS（按业务选择一种并统一）。 |
| **SCORE=0 且未批阅** | 不视为真实得分；**算平均分、通过率等指标时排除**（如仅用 STATUS=4 且 SCORE 非空）。 |
| **时间格式不统一** | 统一解析为 datetime；无法解析的标为缺失；作答时长 = ANSWER_TIME − CREATER_TIME，异常值（如负数、>24h）可截断或标缺失。 |
| **COMMENT 为 HTML** | 清洗阶段可先保留原文；后续做 NLP/关键词时再解析 HTML 取纯文本。 |
| **院系/专业/班级一名多 ID** | 建库时建议保留 ID 为主键，名称做展示；关联时以 ID 为准，或建「院系/专业/班级名称↔标准ID」映射表。 |

### 1.4 可关联表与建库用途

- **学生维度**：`CREATER_LOGIN_NAME` / `CREATER_ID` ↔ 学生基本信息（学号/账号）、选课、成绩表中的学号/账号。
- **课程/教学班维度**：`COURSE_ID`、`CLAZZ_ID`、`WORK_ID` ↔ 选课信息、成绩表、课程表（若有）。
- **组织维度**：`COLLEDGE_ID`、`MAJOR_ID`、`CLASS_ID` ↔ 其他表中的院系/专业/班级 ID，用于统一维度表。
- **时间维度**：`CREATER_TIME`、`ANSWER_TIME`、`REVIEW_TIME` 可做时间轴、学期划分，与成绩、选课学期对齐。

**建议**：建库时为本表设「作业提交事实表」，主键可为 (WORK_ID, CREATER_ID) 或加时间；外键关联学生、课程、教学班、院系等维度表。

**样例一行（便于对照表头）**：

| CREATER_ID | CREATER_NAME | CREATER_LOGIN_NAME | ... | WORK_TITLE | STATUS | FULLMARKS | SCORE | TYPE | CREATER_TIME | ANSWER_TIME | TSTAMP |
|------------|--------------|--------------------|-----|------------|--------|-----------|-------|------|--------------|-------------|--------|
| 146882695 | 同学\|老师 | pjxyqwbk736 | ... | 劳动教育大作业 | 3 | 100 | 0 | 1 | 00:25:20 | 00:52:01 | 00:02:03.100000 |

→ STATUS=3 为待批阅，SCORE=0 属未批阅；时间仅为时分秒，需结合业务判断日期。

## 2. 学生体能考核（学生体能考核.csv）

### 2.1 数据概览与用途

- **粒度**：单条记录 = 某学生在某学期的体能考核结果；**同一学生可有多个学期多行**，形成时间线。
- **用途**：分析体测成绩随学期的变化、院系/专业/班级体质分布；与学号 XH 关联可并入学生画像、学业—体质关联分析。

### 2.2 字段说明与特性

| 字段名 | 说明 | 特性与注意 |
|--------|------|------------|
| **XM** | 学生姓名 | **均为「同学\|老师」脱敏，无真实姓名**；分析时忽略，身份以 XH 为准。 |
| **XH** | 学生学号 | **学号，基本唯一**（同一学生在不同学期会重复出现）。与作业提交记录的 **CREATER_LOGIN_NAME**、基本信息等表的学号为同一标识，**跨表关联用 XH**。 |
| **XB** | 学生性别 | 取值为「男」或「女」。 |
| **NJ** | 年级 | 当前数据多为 **2020级** 或 **2021级**。 |
| **YX** | 院系 | 学院名称，如机电工程学院、光电工程学院。 |
| **ZY** | 专业 | 专业名称，如测控技术与仪器、电子信息科学与技术。 |
| **BJ** | 班级 | 班级名称，**含细分方向**，如「2003014软」（软工方向）、「2004052」；同一班级下可有不同方向。 |
| **BH** | 班号 | 班号，如 200301216、2004052。**与 BJ 区别**：BJ 为班级（含方向细分），BH 为班号；例如 BJ=2003014软、BH=200301216。 |
| **CFBFS** | 查分表分数 | 存在**大量 0**，可能表示成绩未录入或未使用；**参考意义有限**，分析时以 SJCJ、HSCJ 为主。 |
| **SJCJ** | 实际成绩 | 体测实际得分，**分析时优先使用**。 |
| **HSCJ** | 换算成绩 | 换算后的成绩（如百分制或等级折算），**分析时优先使用**。 |
| **XQ** | 学期 | 如「2020-2021学年第2学期」「2021-2022学年第2学期」。**同一学生多行 = 多学期**，可构建**按学期的成绩时间线**，看体测变化。 |
| **TSTAMP** | 时间戳 | 当前数据中**多行为同一值**（如 2025-12-11 14:22:00），应为导出时间，**无区分度，可不用于分析**。 |

### 2.3 数据问题与处理方式

| 问题 | 处理方式 |
|------|----------|
| **XM 无真实姓名** | 不做处理；统一用 XH 作为学生主键，姓名从他表（如学生基本信息）关联获取（若该表有真实姓名）。 |
| **CFBFS 大量为 0** | 视为未录入或弃用；**聚合/建模时以 SJCJ、HSCJ 为主**，CFBFS 可选忽略或单独标注缺失。 |
| **SJCJ / HSCJ 为 0 或异常** | 区分「未录入」与「真实 0 分」；若业务上体测成绩有合理范围（如 0–100），可做范围校验，超出则标缺失或异常。 |
| **同一学生多学期** | 保留多行，用于时间线；去重时按 (XH, XQ) 唯一。 |
| **TSTAMP 无意义** | 不参与时间排序；学期信息以 XQ 为准，如需时间序可解析 XQ 或另从他表取。 |

### 2.4 可关联表与建库用途

- **学生维度**：**XH** ↔ 学生作业提交记录（CREATER_LOGIN_NAME）、学生基本信息（XH）、选课/成绩等表的学号，作为**统一学生主键**。
- **组织维度**：YX、ZY、BJ、BH 与作业表、基本信息等表的院系/专业/班级可做统一维度表（注意 BJ 与 CLASS_NAME 等命名差异）。
- **时间/学期**：XQ 与选课、成绩的学期对齐，用于按学期聚合或画时间线。

**建议**：建库时设「体能考核事实表」，主键 (XH, XQ)；外键 XH 关联学生维度表；分析时重点用 SJCJ、HSCJ，CFBFS 可选。

**样例一行**：

| XM | XH | XB | NJ | YX | ZY | BJ | BH | CFBFS | SJCJ | HSCJ | XQ | TSTAMP |
|----|-----|----|------|------|------|--------|--------|-------|------|------|----------------------|---------------------|
| 同学\|老师 | pjxuqwbj474 | 女 | 2020级 | 机电工程学院 | 测控技术与仪器 | 2004052 | 2004052 | 72 | 14.04 | 29 | 2020-2021学年第2学期 | 2025-12-11 14:22:00 |

## 3. 学生选课信息（学生选课信息.csv）

### 3.1 数据概览与用途

- **粒度**：单条记录 = 某学生在某开课学年/学期选中的一门课（教学班）；同一学生多行 = 多门课或多年多学期，可构成**选课与学业轨迹时间线**。
- **用途**：与年级、选课日期结合刻画**学生成长轨迹**；通过 KCH/KKXN/KKXQ 与成绩、作业提交记录关联，分析选课负荷、课程类别分布、是否重修等。

### 3.2 字段说明与特性

| 字段名 | 说明 | 特性与注意 |
|--------|------|------------|
| **XH** | 学号 | 与作业提交 CREATER_LOGIN_NAME、体能考核/基本信息等表的 **XH 一致**，可**唯一标识学生**，跨表关联主键。 |
| **JXBH** | 教学班号 | 格式如「202420251TS00602801」，可能包含开课学年、学期、课程号、课序号等编码；**目前与作业表 CLAZZ_ID 等对应关系待确认**，可后续通过课程号（KCH）或规则解析与教学班维度关联。 |
| **KCH** | 课程号 | 如 TS006028、SS202043，与成绩表、作业表的 **COURSE_ID/课程** 关联，是**课程维度关联的关键**。 |
| **KXH** | 课序号 | 同一课程不同教学班的序号（如 1、2、37），具体编码规则可后续对照业务；与 JXBH 内部分段可能对应。 |
| **XKZY** | 选课专业 | **多数为空**；非空时出现 1、2、3、4、5 等，含义待业务确认，清洗时可先保留或标为「未知」。 |
| **SFCX** | 是否重修 | **1** = 不重修，**2** = 重修；用于区分初修/重修、学业风险等分析。 |
| **XKRQ** | 选课日期 | 如 2025-02-21、2024-05-22；**重要**，结合学生年级可构建**选课与行为时间线**，用于轨迹推演。 |
| **KKXN** | 开课学年 | 如「2024-2025」「2023-2024」，与成绩、体能等表的学期/学年对齐。 |
| **KCLB** | 课程类别 | 见下表编码（1–25）；**存在空白/空值**，需在清洗时填补（如用「基础课」1 或根据同课程/同院系众数等规则）。 |
| **KKXQ** | 开课学期 | **1** 或 **2**，表示第 1 或第 2 学期。 |
| **TSTAMP** | 时间戳 | 多为数据导出或系统写入时间（如 2025-02-25 16:21:00），**同一批数据常相同**，**分析时以 XKRQ、KKXN、KKXQ 为准**，TSTAMP 可不参与逻辑。 |

### 3.3 数据问题与处理方式

| 问题 | 处理方式 |
|------|----------|
| **KCLB 为空** | 视为缺失。可：① 用**基础课（1）**或固定默认值填补；② 按同课程号（KCH）或同开课学年下的众数 KCLB 填补；③ 若有课程主数据，从主数据补。 |
| **XKZY 多为空或 1–5** | 暂不用于关键逻辑；保留原值，可单独建「选课专业」维度待后续解释。 |
| **XKRQ 缺失或格式异常** | 统一解析为日期；无法解析的标缺失；与 KKXN、KKXQ 一起用于时间线时，缺失行可排除或插补。 |
| **TSTAMP 无业务含义** | 不参与排序与关联；时间维度以 XKRQ、KKXN、KKXQ 为准。 |
| **JXBH 与作业/成绩表关联** | 若作业表为 CLAZZ_ID（纯数字），选课表 JXBH 为长字符串，可先通过 **KCH（课程号）** 与成绩、作业的课程维度关联；JXBH 解析规则确定后再做教学班级关联。 |

### 3.4 可关联表与建库用途

- **学生维度**：**XH** ↔ 作业提交（CREATER_LOGIN_NAME）、体能考核/基本信息/成绩等表的学号，**统一学生主键**。
- **课程维度**：**KCH** ↔ 成绩表课程号、作业表 COURSE_ID/课程名称，用于选课—成绩—作业三联分析。
- **时间/轨迹**：**XKRQ + 年级** 构成选课时间线；**KKXN、KKXQ** 与成绩学期、体能 XQ 对齐，便于按学年/学期聚合。
- **教学班**：**JXBH** 待与教学班维度或作业表 CLAZZ_ID 建立映射后，可做班级/课堂级分析。

**建议**：建库时设「选课事实表」，主键 (XH, JXBH) 或 (XH, KCH, KKXN, KKXQ, KXH)；外键 XH 关联学生，KCH 关联课程维度；KCLB 建议建课程类别维度表。

**样例一行**：

| XH | JXBH | KCH | KXH | XKZY | SFCX | XKRQ | KKXN | KCLB | KKXQ | TSTAMP |
|----|------|-----|-----|------|------|------|------|------|------|--------|
| pjxyqxbj566 | 202420251TS00602801 | TS006028 | 1 | （空） | 1 | 2025-02-21 | 2024-2025 | 7 | 1 | 2025-02-25 16:21:00 |

→ SFCX=1 表示非重修；KCLB=7 为能力素质提升；时间线以 XKRQ、KKXN、KKXQ 为准。

## 4. 学生签到记录（学生签到记录.csv）

### 4.1 数据概览与用途

- **粒度**：单条记录 = 某学生在某活动（ACTIVE_ID）下的签到/出勤状态；同一活动多行 = 多名参与人的记录。
- **用途**：**可作为学生到课率的体现**；结合 ACTIVE_STATE 统计已签/缺勤/迟到/早退/请假等，用于学业行为与出勤分析；与学号关联可并入学生画像。

### 4.2 字段说明与特性

| 字段名 | 说明 | 特性与注意 |
|--------|------|------------|
| **ACTIVE_ID** | 活动 id | 标识某次签到活动；**目前与课程/教学班等具体含义的对应关系待确认**，可先作为活动维度主键。 |
| **ACTIVE_STATE** | 活动状态 | **0未签 1已签 2代签 3请假 5缺勤 7病假 8事假 9迟到 10早退 11签到已过期 12公假**（注意：**3=请假**；数据中存在状态 3，**未出现 4**，统计时按上述编码处理即可）。 |
| **LOGIN_NAME** | 参与人账号 | **即学号/登录账号**，与作业提交 CREATER_LOGIN_NAME、体能/选课/基本信息等表的 **XH 为同一标识**，**做主键与学号关联**。 |
| **ATTEND_TIME** | 创建时间 | 示例为 **1602642284000**，为 **Unix 时间戳（毫秒）**，需除以 1000 后转为日期时间；用于签到时刻、到课时间线。 |
| **INSERT_TIME** | 插入时间 | 格式如「2020/11/21 4:02」，**含义清晰**，可作记录入库时间或活动时间参考。 |
| **ROLE** | 角色 | **1=老师，3=学生**；当前数据多为 3 和空白，基本无老师记录，分析时可按「学生」处理。 |
| **XYID** | 院系 id | **存在空白和 0**；可结合 LOGIN_NAME 从其他表（如学生基本信息、选课、作业提交）查找同账号的院系 id 进行**填补**。 |
| **XYMC** | 院系名称 | **大量空白和 null**（含字符串 `"null"`）；同上，**优先从其他表按账号填补**，而不是直接删行。 |
| **MAJORID** | 专业 id | **大量空白和 0**；分析时可用「仅分析非空/非 0」的策略，或从他表按账号填补。 |
| **MAJORNAME** | 专业名称 | **大量空白**；处理方式同院系，填补或仅用非空数据做分析。 |
| **CLASSID** | (行政)班级 id | **大量空白和 0**；同上，填补或仅分析非空。 |
| **CLASSNAME** | 班级名称 | **大量空白**；同上。 |
| **TSTAMP** | 更新时间 | 多为数据查看/导出时间，**与学生行为分析关系不大**，可不参与建模与排序。 |

### 4.3 数据问题与处理方式（空白值优先填补、慎删）

**原则**：**尽量保留有用数据，不简单删除**。对空白和 null 的处理顺序：① 从本表或其他表按 LOGIN_NAME（学号）查找可用的院系/专业/班级信息**填补**；② 若无法填补，则**仅用非空数据做训练或分析**，不因单列空值整行删除（除非业务要求）。

| 问题 | 处理方式 |
|------|----------|
| **XYID / XYMC / MAJORID / MAJORNAME / CLASSID / CLASSNAME 空白或 null** | **不直接删行**。① 用 LOGIN_NAME 关联学生基本信息、选课、作业提交等表，取该账号的院系/专业/班级进行**填补**；② 无法填补时保留原记录，在后续分析中**仅对非空维度做统计或建模**。 |
| **字符串 "null"** | 与空白统一视为缺失，参与填补逻辑；若展示时需区分，可标为「未知」或留空。 |
| **ATTEND_TIME 为时间戳毫秒** | 清洗时转为标准日期时间（除以 1000 后解析），便于与 INSERT_TIME 及他表时间对齐。 |
| **ROLE 空白** | 当前数据多为学生，可默认填 3（学生）或仅分析 ROLE=3 的记录。 |
| **TSTAMP 无业务含义** | 不参与排序与关联。 |
| **ACTIVE_STATE 3=请假、无 4** | 数据中部分为状态 3（请假），无状态 4；统计到课率时按「已签=出勤」等规则定义，3 归入请假。 |

**写代码时**：对所有涉及院系/专业/班级的空白与 null，统一走「先尝试从关联表填补，再决定是否参与分析」的逻辑，避免因单列空值大面积删行。

### 4.4 可关联表与建库用途

- **学生维度**：**LOGIN_NAME** ↔ 作业提交（CREATER_LOGIN_NAME）、体能/选课/基本信息/成绩等表的 **XH**，**统一学号主键**。
- **组织维度**：XYID、XYMC、MAJORID、MAJORNAME、CLASSID、CLASSNAME 与他表院系/专业/班级关联，**用于填补本表空白**及统一维度表。
- **时间**：ATTEND_TIME（转成 datetime）、INSERT_TIME 可用于到课时间线、按学期/日期聚合。

**建议**：建库时设「签到事实表」，主键 (ACTIVE_ID, LOGIN_NAME) 或加 ATTEND_TIME；外键 LOGIN_NAME 关联学生维度；院系/专业/班级优先通过关联表填补后再写入或分析。

**样例一行**：

| ACTIVE_ID | ACTIVE_STATE | LOGIN_NAME | ATTEND_TIME | INSERT_TIME | ROLE | XYID | XYMC | MAJORID | MAJORNAME | CLASSID | CLASSNAME | TSTAMP |
|-----------|--------------|------------|-------------|-------------|------|------|------|---------|------------|---------|-----------|--------|
| 307328369 | 1 | pjxuqwbj925 | 1602642284000 | 2020/11/21 4:02 | 3 | 12755941 | "null" | 0 | （空） | 0 | （空） | 2025/12/15 4:19 |

→ ATTEND_TIME 为毫秒时间戳；ACTIVE_STATE=1 表示已签，3=请假；ROLE=3 表示学生；院系/专业/班级空时从其他表按 LOGIN_NAME 填补。

### 6.3 数据问题与处理方式

| **DJCJ 为空或为负（-50 至约 -999）** | 题面与样本显示**等级成绩字段质量很差**：多为空或为负数，分布在 -50~-999 之间，**可视为无效/占位信息**。建议：① 主分析与建模中**不使用 DJCJ**；② 如需查看等级信息，可单独建「等级成绩异常说明」，不参与统计。 |

## 8. 学籍异动（学籍异动.csv）

### 8.1 数据概览与用途

- **粒度**：单条记录 = 某学生的一次学籍异动事件（某日期、某类别、某原因，从某院系/专业到某院系/专业）。
- **用途**：通过异动**类别（YDLBDM）与原因（YDYYDM）分析学生发展轨迹与风险**，用于画像（如转专业、出国交流、休学/退学、毕结业等）和学业预警（如因学业问题退学、频繁异动）。

### 8.2 字段说明与特性

| 字段名 | 说明 | 特性与注意 |

| **YDRQ** | 异动日期 | 当前数据中**全部为空白**，无法直接作为时间字段；实际时间可主要依赖审批日期 SPRQ 与生效日期 BY1。 |

| **XZYDM** | 现专业代码 | 异动后专业编码；**有较多空白**。 |
| **XZYFXDM** | 现专业方向代码 | 异动后专业方向编码；**有较多空白**。 |
| **XNJ** | 现年级 | 异动后年级；存在约 23 条空白。 |
| **SFZX** | 是否在校 | 标识学生是否仍在校；当前数据中**大部分为空白**，可结合异动类别/原因推断（如退学/毕结业后为不在校）。 |
| **YDSM** | 异动说明 | 文本说明，**不规范不统一**（如「分流」「根据书院制学生管理规定...」等），可后续做文本归类（规则或 NLP）以辅助解释。 |

| **XBH** | 现班号 | 异动后班号（如 2003013软、2004078 等），也有空白。 |

| **BY2** | 备用 | 全部为空白，可忽略。 |
| **SFSCXKJG** | 是否删除选课结果 | 0/1/空白；表示异动时是否同步删除选课结果。 |
| **SFTBPYFA** | 是否同步培养方案 | 0/1；表示是否同步更新培养方案。 |

### 8.3 数据问题与处理方式

| 问题 | 处理方式 |
|------|----------|
| **YDRQ 全部为空** | 不作为时间字段使用；以 SPRQ（审批日期）和 BY1（生效日期）作为异动时间基准。 |
| **XYXDM/XZYDM/XZYFXDM/XNJ/SFZX 等大量空白** | 不直接删行。① 对于涉及院系/专业/年级的空白，可通过 XH 在基本信息、选课、成绩等表中查找**对应时间点附近**的院系/专业/年级予以填补；② 无法填补时保留记录，仅在「需要这些维度」的分析中筛选非空样本。 |
| **YDSM 异动说明文本不规范** | 不直接依赖原文；可对高频关键词（如「分流」「书院转学院」「退学」「休学」等）做规则归类，将文本映射到少量标准标签，并与 YDLBDM/YDYYDM 交叉校验。 |
| **YXZ/ XB H/ SFZX 存在空白** | YXZ（原学制）和 XBH（现班号）对大部分分析非核心，可直接只用非空记录；SFZX 可通过异动类别/原因推断（如退学/毕结业→不在校），不必强行填补所有空白。 |
| **BY2 全部为空** | 完全忽略，不参与清洗与建库。 |

### 8.4 用于画像与风险分析的设计思路

- **发展规划/画像**：根据 YDLBDM/YDYYDM 将学生分为不同发展轨迹类型，如「转专业/分流型（4/5/6/18/20 等）」「出国交流型（15/18/19）」「高风险（22 因学业问题退学、8/9/28/29 因事/因病退学）」「正常毕结业（17/30）」「学院/班级异动（16/18/31）」等。
- **时间线构建**：使用 SPRQ/BY1 与其他表（成绩、选课、竞赛等）时间对齐，分析异动前后的成绩变化、选课结构变化等。
- **在校状态**：结合 YDLBDM（退学/毕结业等）、YDYYDM（退学/复学等）及 SFZX，构造「是否在校」时间序列，用于过滤建模样本（只保留在某时点仍在校的学生）。

**样例若干**：

- `pjxyqwbj362  …  YDLBDM=4 (转专业/分流), YDYYDM=6 (专业分流), SPRQ=2022/8/16, YDSM=分流, YXZ=4, XBH=2003013软, BY1=2022/8/16`  
- `pjxuqwbj547 …  YDLBDM=18 (学院异动), YDYYDM=31 (学籍从书院转入学院), SPRQ=2021/7/16, YDSM=根据书院制学生管理规定…`  

→ 这类记录可以统一归类为「书院 → 学院」的正常学籍迁移，与风险无直接关系，但对「发展路径」有刻画价值。

## 16. 线上学习平台课程学习行为统计（data2/线上学习平台统计.xlsx）
