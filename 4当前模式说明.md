# 当前模式说明（通俗版，已更新到 mode=8）

## 1. 大类（mode）和小类（subtype）怎么分？

### 1.1 大类 mode（先分）

- 每条“学生-学期”记录先压成 6 个稳定维度分数：
  - `dim_academic`（学业）
  - `dim_attendance_engagement`（出勤/参与）
  - `dim_homework_behavior`（作业行为）
  - `dim_online_learning`（线上学习）
  - `dim_fitness`（体能）
  - `dim_development`（发展成就：奖学金/竞赛/四六级）
- 再用 GMM 在这些维度分数上做软聚类，得到 mode（大类）。
- 最终展示版口径：mode 固定 **K=8**（稳定性优先）。

### 1.2 小类 subtype（再分）

- 在每个 mode 内，再用更细的原始特征（冻结表）做一次 GMM 软聚类。
- 用来把同一大类里不同风格的人再拆开，便于解释。
- 当前每个 mode 下是 4 个 subtype（总计 32 个 subtype）。

### 1.3 subtype_id 怎么看？

- 编码规则：`subtype_id = mode_id * 100 + 子类编号`
- 例如：`203` = `mode 2` 下 `sub 3`；`701` = `mode 7` 下 `sub 1`

### 2.1 各 mode 的核心区别（一句话）

- `mode 1 vs mode 6`：都偏弱，但 `mode 1` 学业更差且波动更明显，风险更高。
- `mode 2 vs mode 7`：都不差，`mode 2` 学业/参与更突出；`mode 7` 更像大多数学生的“均衡状态”。
- `mode 3`：最大特点是“发展成就”明显高（奖学金/竞赛/证书维度更突出）。
- `mode 5`：最突出的差异是“线上学习维度显著偏低”（容易单独识别）。

### 3.1 各 mode 下的小类（subtype）一览（通俗版）

## 4. 对这份结果怎么用

- 给学生：用 mode 短名 + subtype 短名（一句话）就够。
- 给老师/答辩：附上 `subtype_top_features.csv` 做证据支撑。
- 给可视化：用 `student_term_modes_and_subtypes.csv` 做查询与展示主表。

## 5. 查询字段建议（给“按学号查学生”接口）

如果后续要给大模型做个性化解释，建议最少传这些字段：

- 身份键：`XH`, `TERM_KEY`
- 模式层：`mode_id`, `p_mode_`*, `mode_pmax`, `mode_entropy`
- 子类层：`subtype_id`, `p_subtype_*`
- 维度分数：`dim_academic`, `dim_attendance_engagement`, `dim_homework_behavior`, `dim_online_learning`, `dim_fitness`, `dim_development`

这样可以同时支持：

- “你属于哪个模式/子类”
- “你的判断置信度高不高”
- “你主要是在哪些维度上更高/更低”
