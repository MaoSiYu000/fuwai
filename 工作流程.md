## 一、数据预处理阶段（code/pre）

- 各脚本独立：`code/pre/pre_*.py`，可单独运行。  
- 一键运行全部：在项目根目录执行 `python code/pre/run_all_pre.py`（或先 `cd code/pre` 再 `python run_all_pre.py`）。

### 1. 学生基本信息（pre_学生基本信息.py）

- **作用**：得到干净的学生主表，保证学号唯一、政治面貌与出生日期等可分析；为签到、作业等表按学号填补院系/专业提供来源。
- **做法**：删除缺学号行；政治面貌空白用众数填补；出生日期空白可删或保留；按学号去重保留第一条。
- **结果**：清洗前 2500 行，删除 0 行，ZZMMMC 众数填补 2 条，去重后 2500 行；输出 `output/pre/学生基本信息_pre.csv`。

---

### 2. 学生成绩（pre_学生成绩.py）

- **作用**：得到有效成绩记录，便于后续按学期/课程做学业分析。
- **做法**：严格按 `数据集.md` 中的规则：删除缺学号或课程号行；对 KCCJ/BY1/JDCJ 做 0–100、0–100、0–5 范围校验，超出删除；把 BY1 与 JDCJ 同时空或为负的记录删除；DJCJ 负数或空白视为“无效辅助字段”，只做统计不删行。
- **结果**：清洗前 288164 行；删除 KCCJ 越界 3024 行、BY1 越界 40543 行、BY1+JDCJ 同时空/负 5182 行（缺学号/课程号、JDCJ 越界为 0）；DJCJ 无效（空/负）但保留 239415 行；最终清洗后 239415 行，总删除 48749 行；输出 `output/pre/学生成绩_pre.csv`。

---

### 3. 学生选课信息（pre_学生选课信息.py）

- **作用**：得到规范选课记录，便于与成绩、作业按课程与学期关联。
- **做法**：删除缺学号或课程号行；对 KCLB 课程类别先按同一课程号 KCH 的众数进行填补，若整门课均为空再用 1（基础课）填补；XKRQ 解析为日期并新增 `XKRQ_parsed` 列，记录解析成功/失败情况；TSTAMP 不参与逻辑。
- **结果**：清洗前 274695 行，删除缺学号/课程号 0 行；KCLB 按课程众数填补 1058 行、按默认 1 填补 4 行；XKRQ 成功解析 274695 行、失败 0 行；清洗后 274695 行；输出 `output/pre/学生选课信息_pre.csv`。

---

### 4. 学生作业提交记录（pre_学生作业提交记录.py）

- **作用**：得到规范作业/考试提交记录，保证分数≤满分、主键完整。
- **做法**：删除缺 CREATER_LOGIN_NAME 或 COURSE_ID 的记录；删除 SCORE > FULLMARKS 的异常记录；新增 `IS_UNGRADED` 标记字段：当 STATUS=3（待批阅）且 SCORE=0 时标记为 1，其余为 0，便于后续计算平均分时排除未批阅记录；尝试解析 CREATER_TIME/ANSWER_TIME 得到标准时间，并计算简单耗时（秒），用于后续行为分析。
- **结果**：清洗前 49789 行；删除缺主键 0 行、分数大于满分 0 行；标记未批阅记录 IS_UNGRADED=1 共 12150 行；CREATER_TIME 与 ANSWER_TIME 均成功解析的记录 49681 行、至少一端解析失败 108 行；清洗后仍为 49789 行；输出 `output/pre/学生作业提交记录_pre.csv`。

---

### 5. 学生签到记录（pre_学生签到记录.py）

- **作用**：得到规范签到记录，用于到课率与出勤分析。
- **做法**：删除缺 LOGIN_NAME 的记录；将 ACTIVE_STATE 限定在 {0,1,2,3,5,7,8,9,10,11,12} 范围内，其他状态视为非法并删除；ROLE 仅允许 1/3，其它视为非法并删除；ATTEND_TIME 作为毫秒时间戳解析为标准时间 ATTEND_DATETIME；XYMC 中的字符串 `"null"` 统一视为缺失，转为空字符串；此外，利用 `学生基本信息_pre.csv` 按 LOGIN_NAME=XH 填补 XYMC（学院名称）和 MAJORNAME（专业名称），其余 XYID/MAJORID/CLASSID 等字段本阶段不因空白删除，后续可继续按学号跨表填补。
- **结果**：清洗前 748679 行；删除缺账号 0 行、状态非法 0 行、角色非法 0 行；成功解析 ATTEND_TIME 得到 ATTEND_DATETIME 748679 行、解析失败 0 行；利用学生基本信息表填补 XYMC 0 行、MAJORNAME 0 行（当前数据中签到表已有完整学院/专业名称）；清洗后 748679 行；输出 `output/pre/学生签到记录_pre.csv`。

---

### 6. 学生体能考核（pre_学生体能考核.py）

- **作用**：得到规范体测记录，便于按学期做体质与学业关联分析。
- **做法**：删除缺学号 XH 的记录；对 SJCJ、HSCJ 做 0–100 范围校验，超出视为异常并删除；CFBFS 不删，仅作为辅助字段；其他字段（NJ、YX、ZY、BJ、BH、XQ 等）原样保留，供后续按学期和班级做体质分析。
- **结果**：清洗前 4856 行；删除缺学号 0 行、SJCJ 越界 0 行、HSCJ 越界 0 行；清洗后 4856 行；输出 `output/pre/学生体能考核_pre.csv`。

---

### 7. 学科竞赛（pre_学科竞赛.py）

- **作用**：得到规范竞赛获奖记录，便于按学号与时间做画像与统计。
- **做法**：严格处理时间与年级等字段：删除缺 XHHGH 行；删除学院字段中明显错误（如「2022秋季」等）；对 HJSJ 中的 45047 等纯数字、无法解析或月份>12 的日期视为错误并删除；仅年份（如 2022）解析为 该年 1 月 1 日；NJ 统一到“XXXX级”的格式（如 2020、20级→2020级，明显不规范的置空）；ZY 去除 `&nbsp;` 等 HTML 空格并裁剪。
- **结果**：清洗前 2034 行，删除缺学号 0 行、删除学院错误 1 行、删除 HJSJ 纯数字或无法解析日期 11 行，合计删除 12 行；规范 HJSJ 只有年份的记录 0 行、NJ 年级 规范化 954 行、ZY 专业名 1 行；最终清洗后 2022 行；输出 `output/pre/学科竞赛_pre.csv`。

---

### 8. 学籍异动（pre_学籍异动.py）

- **作用**：得到规范学籍异动记录，便于做发展轨迹与风险画像。
- **做法**：删除缺学号 XH 的记录；对 YDLBDM（异动类别代码）和 YDYYDM（异动原因代码）做合法性检查：仅保留题面给出的代码范围（例如 YDLBDM 1,2,3,4,5,11,12,13,14,15,16,17,18，YDYYDM 1–31），超出视为非法并删除；将 SPRQ（审批日期）和 BY1（生效日期）解析为日期，新增 `SPRQ_parsed`、`BY1_parsed` 列，并统计解析成功/失败行数；其它字段（院系/专业/年级、YDSM 文本说明、BY2 等）本阶段不因空白删除。
- **结果**：清洗前 2551 行；删除缺学号 0 行、YDLBDM 非法 0 行、YDYYDM 非法 0 行；SPRQ 解析成功 2551 行、失败 0 行；BY1 解析成功 2551 行、失败 0 行；清洗后 2551 行；输出 `output/pre/学籍异动_pre.csv`。

---

### 9. 线上学习（综合表现）（pre_线上学习综合表现.py）

- **作用**：得到规范线上学习综合表现，便于并入学生画像。
- **做法**：删除缺 LOGIN_NAME 的记录；对 BFB 综合表现分数做 0–100 范围校验，超出视为异常并删除；ROLEID 仅允许 1（老师）、3（学生）、7（管理员），其它值视为非法并删除；院系/专业/班级字段原样保留，用于后续画像分析。
- **结果**：清洗前 2498 行；删除缺账号 0 行、BFB 越界 0 行、ROLEID 非法 0 行；清洗后仍为 2498 行；输出 `output/pre/线上学习（综合表现）_pre.csv`。
